# 用户权限体系 — 实施任务清单

## 任务概览
基于设计文档（design.md）和当前实现状态，完成用户权限体系的完整实施。

**当前状态**: 部分实现（核心组件就绪，功能未完整）
**目标**: 完整实现用户权限体系，包括认证、授权、业务线管理、个人中心等

## 阶段一：安全加固与中间件应用（优先级：紧急）

### 1.1 应用认证中间件
- [ ] 在 `install.go` 中为所有管理路由应用 `AuthMiddleware`
- [ ] 测试所有管理路由的未登录访问（应返回 401）
- [ ] 测试已登录用户访问（应正常通过）

### 1.2 应用角色校验中间件
- [ ] 为 `/api/admin/*` 路由应用 `RequireRoleMiddleware("super_admin")`
- [ ] 为业务线管理路由应用 `RequireRoleMiddleware("super_admin", "line_admin")`
- [ ] 测试角色权限（超管 vs 业务线管理员 vs 普通成员）

### 1.3 应用资源权限中间件
- [ ] 为资源相关路由应用 `RequirePermissionMiddleware`
- [ ] 测试资源隔离（业务线管理员只能操作本线资源）

### 1.4 修复安全问题
- [ ] 检查所有路由是否都有认证保护
- [ ] 检查是否有未授权的 API 暴露
- [ ] 添加 CSRF 保护（如需要）

## 阶段二：完整 API 实现（优先级：高）

### 2.1 业务线管理 API
- [x] 实现 `PUT /api/admin/lines/:id` - 更新业务线（仅描述/状态，不能修改名称）
- [x] 实现 `DELETE /api/admin/lines/:id` - 软删除业务线（标记删除，不物理删除）
- [x] 实现唯一性约束：业务线名称唯一
- [x] 权限控制：仅超管可创建业务线；超管和业务线管理员可修改（仅描述/状态）
- [x] 测试业务线 CRUD 完整流程（软删除、唯一性、权限控制）
- **新要求**：
  - 业务线创建仅超管操作
  - 业务线修改可超管或业务线管理员（仅描述/状态，不能改名称）
  - 软删除：status='deleted'
  - 业务线管理员关联业务线（可多个），不关联应用，由超管设定

### 2.2 用户管理 API
- [ ] 实现 `DELETE /api/admin/members/:id` - 删除用户
- [ ] 实现 `PUT /api/admin/members/:id` - 更新用户信息
- [ ] 实现 `PUT /api/admin/members/:id/password` - 修改密码
- [ ] 测试用户管理完整流程

### 2.3 应用管理 API（business_line_apps）
- [x] 实现 `GET /api/apps?line_id=xxx` - 获取业务线下的应用列表（二级结构：业务线/应用）
- [x] 实现 `POST /api/apps` - 创建应用（业务线管理员可创建本业务线应用）
- [x] 实现 `PUT /api/apps/:app_key` - 更新应用（业务线管理员可修改本业务线应用）
- [x] 实现 `DELETE /api/apps/:app_key` - 删除应用（软删除）
- [x] 实现唯一性约束：业务线内应用名称唯一
- [x] 权限控制：超管可管理所有应用；业务线管理员仅管理本业务线应用
- [x] 资源范围：普通用户只能看到自己所在应用；业务线管理员看到本业务线下全部应用
- [x] 应用列表二级结构显示：业务线 → 应用
- [x] 测试应用管理完整流程
- **新要求**：
  - 业务线管理员可以管理业务线下面的应用（创建、修改信息）
  - 业务线管理员可以把普通用户绑定到应用
  - 应用列表显示二级结构：业务线/应用
  - 应用名称在业务线内唯一
  - 应用管理路由：GET/POST/PUT/DELETE /api/apps（不再使用 /api/admin/apps）

### 2.4 权限管理 API
- [x] 实现 `GET /api/permissions` - 获取权限列表（支持按用户/资源过滤）
- [x] 实现 `DELETE /api/permissions/:id` - 撤销权限（无批量操作）
- [x] 权限控制：超管可管理所有权限；业务线管理员仅管理本业务线权限
- [x] 测试权限管理完整流程
- **新要求**：
  - 权限操作无批量操作（逐个撤销）
  - 超级管理员不需要关联业务线和应用
  - 业务线管理员关联业务线（可多个），不关联应用，由超管设定
  - 普通用户只能查到自己所在应用的范围
  - 业务线管理员可以看自己业务线下全部应用

### 2.5 审计日志 API
- [ ] 实现 `GET /api/admin/audit-logs` - 获取审计日志列表（支持过滤）
- [ ] 实现 `GET /api/admin/audit-logs/:id` - 获取审计日志详情
- [ ] 测试审计日志查看

### 2.6 用户个人中心 API
- [ ] 实现 `GET /api/user/profile` - 获取当前用户资料

- [ ] 实现 `PUT /api/user/password` - 修改当前用户密码
- [ ] 测试个人中心完整流程

## 风险与注意事项

1. **安全风险**：阶段一必须优先完成，否则所有管理 API 都是公开的
2. **数据迁移**：会话持久化可能需要数据迁移（当前内存会话）
3. **向后兼容**：API 变更需要保持向后兼容（如 /api/admin/members 路径）
4. **性能**：权限检查可能影响性能，需要优化查询
5. **测试覆盖**：权限检查逻辑复杂，需要充分测试
